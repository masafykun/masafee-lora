# 個人制作キャラクターの画像生成LoRAファインチューニング：消費者向けGPUによる実験的検討

**著者:** Masato Suzuki (Masafy)
**日付:** 2026年5月22日
**環境:** 自宅GPU PC（NVIDIA GeForce RTX 3060）

---

## 要旨 (Abstract)

本報告では、著者が個人制作したオリジナルキャラクター「マサフィー」（LINEスタンプとして公開済みのレッサーパンダキャラクター）の新規イラストを生成することを目的として、Stable Diffusion 1.5 に対する LoRA（Low-Rank Adaptation）ファインチューニングを行った。学習にはクラウドの有料計算資源を用いず、消費者向けGPUである NVIDIA GeForce RTX 3060（VRAM 12GB）を搭載した自宅PCのみを使用した。既存スタンプ16点から品質基準で13点を選定して学習データとし、(1) 初回学習による実現可能性検証、(2) ベースモデル・学習率・LoRA次元を変化させた6条件の比較実験、(3) エポック数10〜60の追加学習による過学習点の特定、の3段階の実験を実施した。その結果、(a) ベースモデルの選択が出力品質に最も大きく影響し、イラスト指向モデルを用いることで背景・画風の問題が解消されること、(b) 学習エポック数は約30で性能が頭打ちとなり、それ以降は改善せず軽度の過学習傾向を示すこと、(c) 出力品質の上限は学習画像枚数によって規定されること、を明らかにした。本実験を通じて、消費者向けGPU 1台で実用的なキャラクターLoRAの作成が完結可能であることを確認した。

**キーワード:** 画像生成, Stable Diffusion, LoRA, ファインチューニング, キャラクター生成, 消費者向けGPU

---

## 1. はじめに

近年、テキストから画像を生成する拡散モデル（diffusion model）が広く普及し、Stable Diffusion をはじめとするオープンなモデルが個人でも利用可能となった。一方で、特定のオリジナルキャラクターを一貫した見た目で生成するには、汎用モデルそのままでは不十分であり、追加学習が必要となる。

LoRA（Low-Rank Adaptation）は、巨大なモデル全体を再学習する代わりに、低ランクの差分行列のみを学習する手法であり、少ない計算資源と少数の学習画像で特定の概念をモデルに追加できる。これにより、個人が所有するキャラクターの追加学習が現実的になっている。

本研究の動機は実用的である。著者は「マサフィー」というレッサーパンダのオリジナルキャラクターを制作し、LINEスタンプとして公開している（全16種）。このキャラクターの**新しいポーズや場面のイラストを、追加で描かずに生成したい**というのが出発点である。

検討にあたり、当初は有料クラウド計算環境（Google Colab Pro+ 等）の利用も候補としたが、本研究では**自宅にある消費者向けGPU 1台のみ**で完結できるかを検証することとした。具体的な問いは以下の3点である。

- **Q1.** 13枚程度の少数画像で、キャラクターの同一性を保ったまま新規ポーズを生成できるか。
- **Q2.** 出力品質を左右する要因は何か（ベースモデル・学習率・LoRA次元・エポック数）。
- **Q3.** 学習エポック数を増やせば品質は向上し続けるのか。

---

## 2. 関連手法

**Stable Diffusion** は潜在拡散モデル（latent diffusion model）であり、本研究では広く普及している SD 1.5 系列を対象とした。**LoRA** はモデルの重み行列 $W$ に対し低ランク差分 $\Delta W = BA$ のみを学習する手法で、学習対象パラメータを大幅に削減する。学習には kohya-ss 氏による `sd-scripts`（LoRA学習の事実上の標準実装の一つ）を用いた。

---

## 3. 手法

### 3.1 計算環境

| 項目 | 内容 |
|---|---|
| GPU | NVIDIA GeForce RTX 3060（VRAM 12GB, Ampere, compute capability 8.6） |
| CPU / RAM | 20論理コア / 30GB |
| OS | Ubuntu 26.04 LTS |
| 学習フレームワーク | kohya-ss/sd-scripts, PyTorch 2.5.1 + CUDA 12.4 |
| Python環境 | システムPython（3.14）が学習ツール非対応のため、`uv` で Python 3.11 を別途導入 |

なお OS 標準の Python 3.14 では学習ツール群が依存解決できなかったため、軽量Python管理ツール `uv` を用いて Python 3.11 環境を分離構築した。これは最新のディストリビューションで学習環境を構築する際の実務的な留意点である。

### 3.2 学習データ

学習データは、著者制作のLINEスタンプ「マサフィー」全16種を出発点とした。図1に示すとおり、キャラクターはレッサーパンダをモチーフとし、ゴーグル・王冠柄のバンダナ・縞模様の尾といった一貫した特徴を持つ。

16種のうち、以下の基準で3点を除外し、**13点**を学習に用いた。

- 解像度が著しく低い画像：1点
- 他と画風が異なるラフスケッチ調の画像：2点

採用した13点はすべて 1000×1000 px・RGB であり、内訳は顔アップ8点・全身3点・上半身1点・別シーン1点である。

![図1: 学習に用いた13枚](../学習データ_13枚.png)
**図1.** 学習に用いたマサフィースタンプ13点。

### 3.3 キャプション設計

各画像には、学習時に参照されるキャプション（説明文）を付与した。方針は以下のとおりである。

- **トリガーワード**として固有語 `masafee` を全画像の先頭に固定配置した。
- **可変としたい属性**（ポーズ・表情・背景・文字の有無）はキャプションに明記した。これにより、それらは独立した概念として学習され、生成時に制御可能となる。
- **キャラクター固有の不変要素**（ゴーグル、バンダナ等）はあえてキャプションに書かないことで、トリガーワード `masafee` 自体に結びつくようにした。

スタンプには「OK!」等の文字が画像に焼き込まれているため、これをトリミング除去する代わりに「文字が含まれる」旨をキャプションに記述し、モデルが文字とキャラクターを分離して学習するようにした。

### 3.4 学習設定

LoRA学習の基本設定を表に示す。最適化手法は 8-bit AdamW、混合精度は bfloat16、アテキストエンコーダを含む LoRA を学習した。注意機構は PyTorch 標準の SDPA を用い、追加ライブラリ（xformers）への依存を排した。

---

## 4. 実験

3段階の実験を実施した。

### 実験1：初回学習（実現可能性検証）

ベースモデルに標準 SD 1.5（`v1-5-pruned-emaonly`）、解像度512px、LoRA次元32／alpha16、学習率1e-4、バッチサイズ2、16エポック（1040ステップ）で初回学習を行った。

### 実験2：条件スイープ（要因比較）

出力品質を左右する要因を特定するため、以下6条件を自動で学習・比較した。解像度は768px、各30エポック（エポック10/20/30で重みを保存）とした。

- ベースモデル：標準SD1.5 / Counterfeit-V3.0（イラスト指向モデル）
- 学習率：1e-4 / 2e-4
- LoRA次元：16（alpha8）/ 32（alpha16）

各保存点に対し、学習データに存在しない5種の評価プロンプト（手を振る・ジャンプ・コーヒーを持つ・怒り顔・寝る）で画像を生成し、比較グリッドを作成した。

### 実験3：エポック数アブレーション

実験2の最良条件について、学習を**60エポックまで延長**し、エポック30・40・50・60の出力を比較した。乱数シードを固定しているため、エポック30までは実験2と同一の結果が再現され、40〜60が新たに加わる。

---

## 5. 結果

### 5.1 実験1：初回学習は成功

初回学習は RTX 3060 で約9分、VRAM使用量は約3GBであった。生成テストの結果、キャラクターの同一性（ゴーグル・バンダナ・顔のマーキング）は良好に再現され、学習データに無いポーズ（ジャンプ、寝る、宇宙飛行士など）も生成できた。**Q1 は肯定的に確認された。**

一方で2つの弱点が観察された。(i)「白背景」とキャプション指定しても背景が暗色になる、(ii) 元のフラットなスタンプ画風に比べ出力に陰影が強く付く。

### 5.2 実験2：ベースモデルが最も影響する

6条件の比較の結果は明瞭であった（図2）。

![図2a: SD1.5ベース](../スイープ比較/sd15_lr1_d32.png)
![図2b: Counterfeitベース](../スイープ比較/cf_lr1_d32.png)
**図2.** スイープ比較の代表例。上：標準SD1.5ベース。下：Counterfeitベース。各図は縦軸エポック（10/20/30）、横軸が評価プロンプト。

- **標準SD1.5ベース**の3条件：いずれも背景が暗色化し、画風に陰影が強く付く（実験1と同じ弱点）。
- **Counterfeitベース**の3条件：背景が白く整い、フラットな塗りで元のスタンプ画風に近い出力となった。

すなわち、実験1で観察された2つの弱点は、**学習率やLoRA次元の調整ではなく、ベースモデルをイラスト指向のものに変更することで解消された**。学習率・LoRA次元による差は、ベースモデルによる差に比べ小さかった。**Q2 に対し、検討した要因の中ではベースモデル選択が支配的であると結論する。**

最良条件は `Counterfeit + 学習率1e-4 + LoRA次元32`（以下 cf_lr1_d32）とした。

### 5.3 実験3：エポック数は約30で頭打ち

最良条件を60エポックまで延長した結果を図3に示す。

![図3: エポック30-60の比較](../スイープ比較/延長テスト_epoch30-60.png)
**図3.** cf_lr1_d32 をエポック30/40/50/60で比較。各行間の差はわずかである。

エポック30・40・50・60の出力はほぼ区別がつかず、**30エポックの時点で性能はすでに頭打ち（プラトー）に達していた**。エポック50〜60ではむしろ彩度が増し、わずかな過学習（焼き込み）の兆候が見られた。**Q3 に対し、本データ・本設定では学習エポック数を増やしても約30以降は品質が向上しないと結論する。**

### 5.4 最終モデルによる生成

最良モデル（cf_lr1_d32, エポック30）を用いて、学習データに存在しない12種の場面（踊る・ラーメンを食べる・読書・ギター演奏・料理・雨の中を走る等）を生成した（図4）。いずれもキャラクターの同一性とフラットな画風を保ち、実用に足る品質であった。

![図4: 最終モデルによる新規ポーズ生成](../ショーケース_一覧.png)
**図4.** 最良モデルによる新規12ポーズの生成例。

なお「雨の中を走る」では当初、背景が暗く不穏な印象となった。これはプロンプトに背景の明るさを指定しなかったためであり、「明るい・淡い色・水色背景」を明示し、ネガティブプロンプトに「暗い・怖い」を加えることで改善した。シーン依存の属性は明示的に指定すべきという実務的知見である。

## 6. 考察

**ベースモデルの支配性。** 本実験の最大の知見は、出力品質がベースモデルの選択に強く依存することである。キャラクターの画風がフラットなイラストである場合、写実指向のベースモデルでは画風が衝突し、背景や陰影に悪影響が出る。学習対象の画風に合ったベースモデルを選ぶことが、ハイパーパラメータ調整よりも優先される。

**過学習とエポック数。** 13枚という少数データでは、約30エポックで学習が飽和する。それ以上の学習は品質を向上させず、過学習（学習画像の暗記、新規ポーズ生成能力の低下、背景・色の固定化）のリスクを高める。「長時間学習すれば良くなる」という直感は誤りであり、適切なエポック数はデータ量に応じて存在する。

**データ量が品質の上限を決める。** エポック数が頭打ちである以上、さらなる品質向上の主たる手段は学習画像の追加である。13枚という枚数自体が到達可能な品質の上限を規定している。

**消費者向けGPUの十分性。** 各学習は RTX 3060 で9〜40分、VRAM使用量は3〜8GBであった。SD1.5 のキャラクターLoRA作成において、有料クラウド計算環境は必須ではなく、消費者向けGPU 1台で実験的な反復を含めて完結できる。

---

## 7. 限界と今後の課題

- **評価の主観性。** 本研究の評価は比較グリッドの目視によるものであり、定量指標（CLIP類似度、FID等）を用いていない。
- **データ規模。** 学習画像は13枚、対象キャラクターは1体であり、一般化には限界がある。
- **文字の焼き込み。** 学習画像中の文字はトリミング除去せずキャプション記述で対処した。除去した場合の効果は未検証である。
- **スイープの分散。** 各条件は単一シードで評価しており、シード由来の変動は捉えていない。

今後の課題として、学習画像の追加による品質向上、定量評価指標の導入、SDXL等のより新しいベースモデルでの検証が挙げられる。

---

## 8. 結論

著者制作のオリジナルキャラクター「マサフィー」を対象に、消費者向けGPU（RTX 3060）のみを用いて画像生成LoRAのファインチューニングを行った。13枚の学習画像から、キャラクターの同一性を保ったまま新規ポーズを生成できることを確認した（Q1）。6条件の比較実験により、出力品質はベースモデルの選択に最も強く依存することを示した（Q2）。さらにエポック数10〜60のアブレーションにより、約30エポックで性能が頭打ちとなり、それ以上の学習は品質を向上させないことを実証した（Q3）。本研究は、個人が所有するキャラクターの追加学習が、有料クラウド資源を用いずとも消費者向けGPU 1台で実用的に完結可能であることを示すものである。
