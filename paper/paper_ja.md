# 個人制作キャラクターの画像生成LoRAファインチューニング
## ——消費者向けGPUを用いた少数データ条件下での実験的検討——

**著者:** Masato Suzuki (Masafy)
**日付:** 2026年5月22日

---

## 要旨

本報告では、著者が個人制作したオリジナルキャラクター「マサフィー」（LINEスタンプとして公開済みのレッサーパンダキャラクター）の新規イラストを生成することを目的として、潜在拡散モデル Stable Diffusion 1.5 に対する低ランク適応（Low-Rank Adaptation; LoRA）に基づくファインチューニングを行った。学習にはクラウドの有料計算資源を用いず、消費者向けGPUである NVIDIA GeForce RTX 3060（VRAM 12GB）を搭載した自宅PCのみを使用した。既存スタンプ16点から品質基準に基づき13点を選定して学習データとし、(1) 初回学習による実現可能性検証、(2) ベースモデル・学習率・LoRAランクを変化させた6条件の比較実験、(3) 学習エポック数を10から60まで変化させたアブレーション、の3段階の実験を実施した。その結果、(a) ベースモデルの選択が出力品質に最も大きく影響すること、(b) 学習エポック数は約30で性能が頭打ち（プラトー）に達すること、(c) 出力品質の実効的上限が学習画像枚数によって規定されること、を実験的に明らかにした。

**キーワード:** 画像生成, 潜在拡散モデル, Stable Diffusion, 低ランク適応, LoRA, ファインチューニング, キャラクター生成, 消費者向けGPU

## 1. はじめに

近年、テキスト記述から画像を生成する拡散モデル（diffusion model）が急速に発展し、Stable Diffusion をはじめとするオープンウェイトのモデルが個人レベルでも利用可能となった。これらは膨大かつ多様な画像コーパスで事前学習されており、汎用的な画像生成能力を有する。しかし、事前学習分布に含まれない*特定の*オリジナルキャラクターを、同一性（identity）を保持したまま一貫して生成することは、汎用モデルそのままでは原理的に困難である。これは、当該キャラクターに対応するモードが事前学習分布中に存在しない、あるいは極めて低密度であることに起因する。したがって追加学習（fine-tuning）が要請される。

全パラメータを更新する素朴なファインチューニングは、(i) 数十億規模のパラメータ更新コスト、(ii) 少数データに対する破滅的忘却・過適合、という課題を抱える。低ランク適応（LoRA）は、事前学習済み重みを凍結し低ランクの差分行列のみを学習することでこれらを緩和し、少ない計算資源と少数画像で特定概念をモデルに注入することを可能にする。

本研究の動機は実用的である。著者はレッサーパンダを主題としたオリジナルキャラクター「マサフィー」を制作し、スタンプ（全16種）として公開している。出発点は、未収録のポーズ・場面のイラストを追加の手描きなしに生成したいという要求である。当初は有料クラウド環境も候補としたが、本研究では自宅の単一の消費者向けGPUのみで完結可能かを検証する。研究設問は以下の3点である。

- **Q1.** 13枚程度の少数参照画像から、キャラクター同一性を保持した新規ポーズ生成が可能か。
- **Q2.** 出力品質を律速する要因は何か（ベースモデル・学習率・LoRAランク・エポック数）。
- **Q3.** 学習エポック数の増加は品質を単調に改善するか。

## 2. 準備：理論的背景

### 2.1 拡散確率モデル

拡散モデルは、データに段階的にガウス雑音を付加する*前向き過程*と、それを逆にたどる*逆過程*からなる潜在変数生成モデルである。データ $x_0\sim q(x_0)$ に対し、前向き過程は分散スケジュール $\{\beta_t\}_{t=1}^{T}$ を持つマルコフ連鎖

$$q(x_t\mid x_{t-1}) = \mathcal{N}\big(x_t;\,\sqrt{1-\beta_t}\,x_{t-1},\,\beta_t\mathbf{I}\big)$$

として定義される。$\alpha_t:=1-\beta_t$、$\bar\alpha_t:=\prod_{s=1}^{t}\alpha_s$ と置くと、任意の時刻 $t$ の状態は閉形式

$$x_t = \sqrt{\bar\alpha_t}\,x_0 + \sqrt{1-\bar\alpha_t}\,\epsilon,\qquad \epsilon\sim\mathcal{N}(0,\mathbf{I})$$

で直接サンプリングできる。逆過程は雑音予測ネットワーク $\epsilon_\theta$ で表現され、学習は雑音予測損失

$$\mathcal{L}(\theta)=\mathbb{E}_{x_0,\epsilon,t}\big[\lVert\epsilon-\epsilon_\theta(x_t,t,c)\rVert_2^2\big]$$

の最小化として行う。$c$ は条件付け変数（本研究ではテキスト埋め込み）。

### 2.2 潜在拡散モデルとアーキテクチャ

Stable Diffusion が採用する潜在拡散モデル（LDM）は、変分オートエンコーダ（VAE）のエンコーダ $\mathcal{E}$ により画像を低次元潜在表現 $z_0=\mathcal{E}(x_0)$ に圧縮し、潜在空間で拡散過程を実行する。生成された潜在変数はデコーダ $\mathcal{D}$ で画素空間へ復元される。雑音予測器 $\epsilon_\theta$ には U-Net が用いられ、内部の*交差注意*（cross-attention）機構を介してテキスト条件 $c$ が注入される。$c$ はテキストエンコーダ（CLIP のテキスト塔）が文を埋め込んだ系列である。

### 2.3 条件付き生成とclassifier-free guidance

推論時には classifier-free guidance（CFG）が用いられる。条件付き予測と無条件予測 $\epsilon_\theta(x_t,t,\varnothing)$ を併用し、ガイダンス強度 $s\ge1$ で

$$\tilde\epsilon_\theta=\epsilon_\theta(x_t,t,\varnothing)+s\big(\epsilon_\theta(x_t,t,c)-\epsilon_\theta(x_t,t,\varnothing)\big)$$

と外挿する。本研究では $s\approx7$ とした。

### 2.4 低ランク適応（LoRA）

事前学習済み重み行列を $W_0\in\mathbb{R}^{d\times k}$ とする。LoRA はこれを凍結し、適応後の重みを

$$W' = W_0 + \Delta W,\qquad \Delta W=\frac{\alpha}{r}\,B A$$

と表す。$B\in\mathbb{R}^{d\times r}$、$A\in\mathbb{R}^{r\times k}$、ランク $r\ll\min(d,k)$。$\alpha$ はスケーリング係数で実効適応強度は $\alpha/r$ に比例する。学習対象は $A,B$ のみ（$r(d+k)$ パラメータ）で、全微調整の $dk$ に比して大幅に少ない。$A$ をガウス初期化、$B$ を零初期化することで学習開始時 $\Delta W=0$ が保証される。本論文では $r$ を「LoRA次元」、$\alpha$ を「alpha」と呼ぶ。

### 2.5 用語整理

| 用語 | 説明 |
|---|---|
| EMA | 指数移動平均。重みの平滑化版。`emaonly` 版はこれを格納。 |
| バケット | 異なる縦横比の画像を解像度群に振り分ける機構。 |
| 勾配チェックポイント | 中間表現を再計算しVRAMを削減する技法。 |
| 混合精度(bf16) | 低精度演算を併用し速度と省メモリを得る方式。 |
| SDPA | PyTorch標準の高速注意計算実装。 |
| トリガー語 | 対象概念に束縛させる固有語。 |

## 3. 問題設定

対象キャラクターを $\mathcal{C}$、参照画像集合を $\mathcal{X}=\{x^{(1)},\dots,x^{(N)}\}$（$N=13$）とする。各画像にはトリガー語 $\tau$（`masafee`）を含むキャプション $c^{(i)}$ が付与される。目標は、事前学習済み $\epsilon_\theta$ を凍結したまま LoRA 差分 $\Delta\theta$ を学習し、

$$\Delta\theta^{\star}=\arg\min_{\Delta\theta}\ \mathbb{E}_{i,\epsilon,t}\big[\lVert\epsilon-\epsilon_{\theta+\Delta\theta}(z_t^{(i)},t,c^{(i)})\rVert_2^2\big]$$

を解くことである。$z_t^{(i)}$ は $\mathcal{E}(x^{(i)})$ に前向き過程を適用した潜在変数。$N$ が小さいため本問題は少数ショット条件にあり、$\Delta\theta$ の表現力（$r$）と学習量（エポック数）が、汎化–過適合トレードオフを通じて品質を律速する。

## 4. 手法

### 4.1 計算環境

| 項目 | 内容 |
|---|---|
| GPU | NVIDIA GeForce RTX 3060（VRAM 12GB, Ampere） |
| CPU / RAM | 20論理コア / 30GB |
| OS | Ubuntu 26.04 LTS |
| 学習フレームワーク | kohya-ss/sd-scripts, PyTorch 2.5.1 + CUDA 12.4 |
| Python環境 | `uv` で Python 3.11 を別途導入（標準の3.14が非対応） |

### 4.2 学習データと選定基準

学習データは著者制作のスタンプ「マサフィー」全16種を母集団とした（図1）。キャラクターはレッサーパンダを主題とし、ゴーグル・王冠柄バンダナ・縞模様の尾という一貫した識別特徴を持つ。母集団から、解像度が著しく低い1点、画風が乖離するラフスケッチ2点を除外し、**13点**を学習集合 $\mathcal{X}$ とした。すべて 1000×1000 px・RGB。

![図1](../学習データ_13枚.png)

### 4.3 キャプション設計とトリガー語束縛

固有語 $\tau=$`masafee` を全キャプション先頭に固定し、可変にしたい属性（ポーズ・表情・背景・文字の有無）をタグとして明示した。明示された属性は独立の概念として学習され制御可能となる。一方、キャラクター固有の不変要素（ゴーグル等）はあえて記述せず、残差的に $\tau$ へ束縛させた。焼き込み文字は除去せず「文字を含む」とキャプションに明記して分離学習させた。

### 4.4 学習設定

最適化は 8-bit AdamW（最適化器状態を量子化しVRAM削減）、混合精度 bfloat16、注意計算は SDPA を用いた。勾配チェックポイント・アスペクト比バケットを併用し、データ拡張として左右反転を適用した。

## 5. 実験

**実験1（実現可能性検証）:** 標準SD1.5、解像度512px、$r=32$/$\alpha=16$、学習率 1e-4、16エポック（1040ステップ）。

**実験2（要因比較スイープ）:** ベースモデル{標準SD1.5, Counterfeit-V3.0}×学習率{1e-4, 2e-4}×LoRA次元{16, 32}の6条件を、768px・各30エポックで学習・比較。エポック10/20/30を保存し、学習集合外の5プロンプトで比較。

**実験3（エポック数アブレーション）:** 最良条件を60エポックまで延長し、エポック30/40/50/60を比較（乱数シード固定）。

## 6. 結果

### 6.1 実験1：初回学習による実現可能性

RTX 3060 で約9分、VRAM約3GB。キャラクター識別特徴は良好に再現され、学習集合に無いポーズも生成できた（**Q1を肯定的に確認**）。一方、(i) 白背景指定でも背景が暗色化、(ii) 陰影が過剰、という2弱点を観察した。

### 6.2 実験2：ベースモデルの支配性

6条件の比較（図2）は明瞭であった。標準SD1.5ベースは背景暗色化・過剰陰影を示し、Counterfeitベースは白背景・フラット彩色で元画風に近い出力を与えた。実験1の弱点は、学習率やLoRA次元ではなく**ベースモデルの変更で解消された**。**Q2に対し、検討要因のうちベースモデル選択が支配的**と結論する。これは LoRA が*差分*のみを学習し、画風的事前分布がベースモデルに規定されるという構造から理論的にも整合的である。最良条件を cf_lr1_d32（Counterfeit + lr 1e-4 + 次元32）とした。

## 7. 考察

**ベースモデルの支配性。** LoRA は $\Delta W$ を学習するに過ぎず、画風的事前分布は凍結された $W_0$ に保持される。対象画風と事前分布が乖離する場合、有限ランクの $\Delta W$ では画風を完全には上書きできない。対象画風に整合するベースモデルの選択はハイパーパラメータ調整に優先する。

**過適合とエポック数。** 13枚の少数データでは、最適化ステップ総数が閾値を超えると $\Delta\theta$ が学習集合の特異な構成を記憶し始める。実験3はこの発現点がエポック30近傍にあることを示した。

**データ量による品質上限。** エポック数がプラトーに達する以上、さらなる品質向上の主たる自由度は学習画像の追加にある。$N=13$ という標本数自体が到達可能性能の実効上限を規定する。

**消費者向けGPUの十分性。** 各学習は RTX 3060 で9–40分、VRAM 3–8GB。SD1.5規模のキャラクターLoRA作成に有料クラウドは必須でない。

## 8. 限界と今後の課題

評価は比較グリッドの目視による定性評価で、CLIP類似度やFID等の定量指標を用いていない。学習集合は単一キャラクター・13枚であり一般化に制約がある。焼き込み文字はキャプション記述で対処し、除去との比較対照は未実施。スイープ各条件は単一シード評価でシード分散を捉えていない。今後は学習画像の拡充、定量指標の導入、SDXL等での検証が課題である。

## 9. 結論

著者制作のキャラクター「マサフィー」を対象に、消費者向けGPU（RTX 3060）のみで画像生成LoRAのファインチューニングを行った。13枚の少数参照画像でキャラクター同一性を保った新規ポーズ生成が可能であること（Q1）、出力品質はベースモデルの選択に最も強く依存すること（Q2）、約30エポックで性能がプラトーに達すること（Q3）を、再現可能な手順とともに実験的に示した。
